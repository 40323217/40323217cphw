CP筆記

利用Git和Github來達到版次管理，並且在近端測試再把資料推送至遠端，在電腦輔助下把重複性的動作藉由批次檔達到我們的需求。第一週重點: git add . -> git commit -m "推送依據" -> git push origin master(非常重要,推送至起初的master) 第二週重點: 1.git clone 網址 -> revel貼上(除README.md) -> push回去遠端 -> 在推送前要先建分支 -> git branch 分支 -> git checkout 分支名子 -> revel-gh-pages 覆蓋 ->推回去所建立的分支 ->利用leo進行檔案更改. 補充筆記: 1. git config --global.user "名子" 和 git config --global.email"輸入信箱" 這兩點是要確認身分 2. pip install pelican markdown 是下載模組.

自評分數75分

我發覺在git branch (git checkout) 這點很重要在推送資料時才知道是推送至分支還是master因為我在這點花了很多時間才弄懂，master檔案要推到master，分支的要推到分支，都要非常清楚自己在每個步驟的重要性一步一步的達到老師的要求，更重要的是英文因為很多第一首的資料都是以英文呈現另外網路上資料都是以英文呈現，看不懂就沒辦法知道解決方法。
